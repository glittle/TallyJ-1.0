Instructions
Version 1.70 - January 2011
  1. Definitions
    Election
    Scheduled event where voters submit ballots
    Ballot
    A list of names, submitted by a Voter. Usually a piece of paper. Each ballot consists of a specific number of votes (e.g. 9 names for Assembly elections).
    Vote
    Name of one individual. A vote is written on a ballot.
    Community List
    For the purposes of elections (and TallyJ), a community is the group of people who are eligible to vote and be voted for in the election. This usually consists of the adult Bahá'ís in good-standing living in a particular geographic area. If others (children, youth, ineligible adults, etc.) are included in the community list, they will not be seen in TallyJ except for the complete community reports.
    Electee
    A person who has been voted for in an election
    Elected
    An electee who has received enough votes to rank in the top 9 (for Assembly elections) positions
    Voter/Elector
    A person who is permitted to submit a ballot (e.g. adults in community or delegates)
    Tellers
    Individuals tasked to count votes cast and report the results to the community
    Spoiled Ballot
    A ballot not containing the required number of distinct votes. All votes on the ballot are ignored.
    Spoiled Vote
    A vote that is not readable or is for someone who is ineligible to receive a vote. The rest of the votes on the ballot are still counted.
    Ineligible
    A person who is not an adult member in good standing of the community
    Tally
    The act of counting the votes in an election.
    TallyJ
    A computer program designed to assist tellers in the tallying of ballots in a Bahá'í election.

  3. Install TallyJ
    Updated for version 1.70

    TallyJ is a set of "web pages" that are copied to your computer and run from there. They do not need or use a connection to the internet or a web server. The only requirements are a Windows XP, Vista or 7 computer with Internet Explorer 8 and MS XML Core Services 6 (already installed on most computers).

    To install TallyJ, download the most current ZIP file from the "Downloads" section of the TallyJ Code site. The contents of the ZIP file must be extracted into a folder on your computer. Later, if you need to install an updated version of TallyJ, the new files can be extracted to the same folder and overwrite the existing files. (Your custom data files will not be affected.)

    When you run TallyJ, it will tell you if you need to install MS XML Core Services. If you do, you will find a link to it in the "Links" section of the TallyJ Support Page on Facebook.

    To run TallyJ, simply double-click on the "TallyJ.hta" file in the folder where the TallyJ files were extracted to. If you are asked for confirmation that you want to open it, please provide it. A shortcut to this file could be created and moved to your Desktop or Quick Launch area for easy access later.

    On election day, all computers that are networked together can use a common set of TallyJ files. However, if TallyJ will be used on computers that are not connected together, repeat these steps on each computer (preferably, before the election).

  4. Practice using TallyJ

    After opening the TallyJ.hta files for the first time, click the Files button at the top. In the list of files shown, double-click "SampleElection" and "SampleCommunity". TallyJ will now use these two files on each page and each time you use TallyJ until you change it here.

    The Election page is where the details of the election are entered.

    The Community page is where the names of people in the system can be viewed and modified.

    The Ballots page is for the entry of ballots and votes. See the section below about entering the ballots into TallyJ for details.

    The Reports page provides a variety of reports about the ballots entered. Try viewing each report and understand what each shows.

    The Save button appears whenever some information needs to be saved. The Save button sometimes does not appear until the cursor is moved out of the entry box, or you click anywhere on the page. Caution: pressing the F5 key at any time will reload the application and if there was a Save needed, the unsaved data will be lost.

    On each page, some words have one letter underlined (like that). Holding the ALT key while typing that letter or number will click the button or move to that spot on the page.

    To close TallyJ, simply close the window. (As long as the Save button is not visible, all ballots and votes are saved to the current Election file.)

    It would be prudent to include a "computer-savvy" person among the tellers to ensure that computer problems do not delay the tallying process!

  5. Prepare a Community file

    When entering the names appearing on a ballot, data entry is made much faster if an accurate Community List has been loaded ahead of time. This list should contain the names of all the eligible members of the community.

    The Community file is an XML file in ANSI format (not UTF or Unicode) that is structured in a particular way. It could look something like:

    <Community Name="Calgary">
      <Person LName="Chang" FName="Francisco"/>
      <Person LName="Danúe" FName="Änj"/>
      <Person LName="Smith" FName="John" AKAName="Johnny"/>
      <Person LName="Smith" FName= "John" AKAName= "JohnA." BahaiId= "W20323"/>
      ...
    </Community>

    The actual file has more attributes on the first line and some extra lines above that. A template community file is supplied. You should copy that file, then edit the new file and insert the <Person> lines for your community. If you have a list of names in something like MS Excel, you can use Excel to format the names as shown and then copy them into your new file.

    Tools to assist in creating the list may be available on the TallyJ web site.

    Each <Person> line must include the LName and FName attributes. If there are two or more people with the same name, an AKAName can also be supplied to distinguish between them and shows on the screen as "Other Info". If desired, a BahaiId can be provided, but this is in no way required by TallyJ. The Tie Break Resolution is not required and is generally entered only when there is a tie-vote at the election.

    If a Community file is not completely filled beforehand, it can be added to as the ballots are entered. If the Community file is empty when starting the tally, a setting on the Ballot page will let you choose to show "Add to List" for all ballots.

    Usually, during an election, the list of people may be reviewed and adjusted by the tellers. Use the Community page to add, correct and remove names from the list. If this is done after some ballots have been entered, all ballots with the changed person will need to be updated to match the new name.

    The Community list can be printed from the Reports page in a number of different formats. This can be used to produce paper lists for use by electors and tellers at the election.

    If more than one computer will be used to enter the ballots and they are not sharing a common Community file, and a Community file is not prepared, extreme caution must be taken to ensure that one individual's name is not entered in two different formats or spelling. If the two names are not precisely identical, TallyJ will consider them to be two separate people and report the results as such, splitting the votes between the two variations of the name.

    Load the Community file into TallyJ from the Data folder using the Files page:

    1. Copy the file into the Data folder
    2. Click the "Files" button in TallyJ. You should see "Files in the Data Folder"
    3. Your file should be in the list near the center of the screen. You may need to click "Refresh List".
    4. Click on the file in the list. A button called "Load Community File" will appear if the file is valid.
    5. Click the "Load Community File" button. This will load that file and use it as the community file.
  6. Prepare the Election File

    An Election file is created on the Files page. Name the new file according to the election being recorded, e.g. "LSA 2003.xml". After creating the file, load it and go to the Election page to configure the contents of the file.

    Before the first ballot is entered, the Election file must be told what to expect. On the Election page of TallyJ, the number of votes to appear on each ballot is selected. For Assembly elections, "Number to Elect" should be set to 9. For Unit or other elections, choose the required number. The other configuration information can be added or adjusted later.

    If more than one computer will be used to enter the ballots, a separate Election file must be prepared for each computer. One file should be considered the "master" file. After the ballots are entered, the "other" election files will be merged into the master file for counting and reporting. The configuration details of the master file should be carefully set. For the other files, only the "Code for this Computer" detail needs to be set. Each file must have a unique code. This is used to "tag" the ballots being entered with a unique Id. Typically, the master election file will be created, loaded and the election information set, and saved; then that file is duplicated to create all the other Election files; then each of the new Election file is loaded and the "Code" is changed and saved.

    For instance, if 5 laptops will be networked together and all accessing a folder on computer M, the TallyJ files are installed on computer M with a Community file (Community.xml) and a master Election file (ElectionMaster.xml). In the ElectionMaster.xml, the "Code for this Computer" is set to, say, M. Four duplicates of the Election file are created called ElectionA, ElectionB, ElectionC and ElectionD. Each Election file is then loaded and the "Code for this Computer" is changed to match the file name: A, B, C or D. Later, for reporting, these 4 files will be merged with the master file.

    Election files are stored in XML format in the Data folder. At any time, they can be manually copied to a backup location, such as a memory stick or another computer, and archived for safety.

  7. Technical Alterations

    Ballot Size - the number of names that can appear on a ballot for Assembly elections is usually 9. For bi-elections, the number is smaller. For Unit Elections, the number to be elected may be more than 9 for larger communities. TallyJ currently sets that limit at 30. If you need to increase that number, two changes need to be made in the election.xsd file: the NumberToElect attribute and maxOccurs in the Ballot/Vote element.

  8. Setup the Computers at the Election Site

    On the day of the election, the tellers' computers must be setup at the election site.

    A suggested setup includes:

    • As many computers as needed, about 1 per every 50 ballots expected
    • Two tellers for each computer
    • One printer, attached to the "master" computer
    • A network router or hub and wiring to connect all computers together. A wireless router can be used if the computers/laptops support it.
    • Enough power bars to supply all computers, monitors, hubs, printers, etc.
    • Extension cords long enough to supply power to the location where the tellers will be counting votes

    Assistance in setting up the computers can be provided by non-tellers, such as the people who may be lending their computers during the election and/or local computer "experts".

    The "master" computer is where the printer is attached and where the data files are stored and collected. This should typically be the fastest or most capable of the computers. Be sure to test printing to the printer before leaving home to verify that the correct printer drivers are installed, etc.

    If you have the equipment to network the computers, do so - it makes tallying the votes faster and easier. Once connected, be sure that all the other computers can connect to and view the TallyJ folder on the "master" computer. This may require setting up new login IDs or sharing existing IDs. Be sure that at least one teller is aware of all the IDs and passwords needed to establish the required network connections.

    Arrange the computers so that two people can be in front of each computer - one typing, and the other reading the names off the ballot papers, writing the Ballot Id on each ballot paper, and verifying the names as they are being typed. If only one teller is at the computer, the process still works, but is a little slower.

  9. Load the Files into TallyJ

    On the Files page of TallyJ, the files currently in the Data folder are listed.

    If the computers are networked, they can and should share a common TallyJ folder. All of them can run the same TallyJ.hta file. A separate copy of the election file should be used for each computer. From each computer, open TallyJ.hta that is located in the shared folder. Then load the common Community file and the computer-specific Election file.

    If the "Code for this Computer" configuration has not been set, do so before entering any ballots. Each computer MUST have its own code and Election file.

  10. Collect the Ballots

    The procedure used to collect the ballots at the election is not prescribed by TallyJ. Each community and country may have their own prescribed methods for doing this.

    As an example of how this might be done...

    Completed ballot slips are placed into unmarked envelopes by voters. At a central location, the ballots are placed in two types of ballot boxes (Present and Dropped Off) and the tellers mark on a community list beside the name of each voter whether they 1) voted in person (Present), or, 2) their sealed ballot was delivered by mail or hand (Dropped Off ).

    When the time for voting is finished, the ballot boxes are opened in private by the tellers, the ballots are removed from the envelopes, and a careful count is made of all the ballots. The totals are compared to the marked up community lists to ensure that all ballots, whether Present or Dropped Off, are accounted for. Collecting the ballots into stacks of 10 may help the counting process. (Important: When opening the ballot envelopes, if a ballot is found to be torn or separated by name, be sure to tape them together so that the ballot remains as a single unit.)

    Once the counts are confirmed, the tellers can begin entering the ballots into TallyJ on the computer(s).

    (The sample process described above is not an officially prescribed process, just an example of what one community has done.)

  11. Enter the Ballots into TallyJ

    The paper ballots being counted should remain intact throughout the tallying process. When a system like TallyJ is not being used, tellers have often cut ballots into individual votes to facilitate manual collating and counting. This should NOT be done when using TallyJ. (In fact, any ballots received on more than one piece of paper must be taped together to form a single ballot!)

    To enter a ballot in TallyJ, click the Add New button on the Ballots page. A new identification (Id) number is immediately created and displayed near the top of the page. This Id consists of the unique "Code" for this computer followed by a number, e.g. "A.12". Write this Id on the paper ballot to make review and checking possible. All tellers should write the ballot Ids in the same location on the paper, for instance, in the top-right corner. Using a red pen is preferred.

    For each name on the ballot, begin typing the name into the "Search" column. As you do, the Community list in the Names area will display the all the names matching what has been typed so far. Continue typing until the correct name can be seen. Press the Up or Down keys to move the highlighting in the List, and when the correct name is highlighted, press Enter.

    When Enter is pressed, the complete name from the list is automatically copied into the Vote. Repeat for the next name on the ballot.

    • If too many names have been put on the ballot, click "Too Many Names?" at the top. If desired, the names on the paper can be entered into TallyJ.
    • If too few names have been entered, leave the last vote line(s) blank. After clicking "Save Ballot", then click the "Save Invalid Ballot" to save the ballot. The ballot status will be set to "Too Few" and not be included in the final count.
    • If a name on the paper is not readable, or is for a person known not to be in the Community, click the "S" button on the line.
    • If the name is valid, but is not yet in the Community list, click the "+" button then enter the full name carefully in the spaces provided. The name will be added to the local Community list when the "Check & Save Ballot" button is clicked. If using a common Community file between more than one computer, everyone else should immediately click the "R" beside the Community name in the top right corner of their screen. If not using a common Community file, carefully ensure the same name is entered identically on the other computers, or copy the updated Community file to the other computers.
    • (If a person voted for is an adult in the community, but inelegible for election, you must update the person's record in the Community list to ensure it is marked as ineligible or is deleted from the file.)
    • If a name is repeated twice on one ballot, you should enter the names exactly as seen on the paper; when the ballot is checked, the status will be set to "Dup Name".

    (Remember, throughout TallyJ, some letters in some words are underlined to indicate keyboard shortcuts. For instance, if a buttons says "Add New", then pressing ALT-A will press the button or go to that spot.)

    If you start the ballot entry without a Community List loaded, you can create it as you enter ballots (as described above). In this case, you should check the box beside "Default: Add to Community List". New ballots will start with blank spaces available for entering names that need to be added to the List.

    In summary, try to enter what is seen on the paper as closely as possible into TallyJ and write the ballot Id on the paper ballot.

  12. Merge the Election files (if needed)

    If more than one computer is being used to enter ballots, the "other" Election files must be merged into the "master" Election file before obtaining final reports.

    If a common folder on a "master" computer is not being used, start by copying the "other" Election files to the computer and folder where the "master" Election file is. This can be done via diskette or memory stick if no other way is available. (The files are very small and hundreds could fit on one diskette.)

    Once the files to be merged are in the same folder as the master file, go to the Files page on the master computer. The file will be listed (click "Refresh List" if needed); select it, then click "Merge with Current Election". The new ballots in that file will be added to the Master file.

    If the "other" Election file being merged has been merged previously, only new ballots will be merged. The Ballot Id is used to determine which have been merged. It is therefore safe to merge the same file files multiple times as the counting proceeds.

  13. View and/or print the Election Results report

    The Reports page can be used at any time to display the current election results.

    Before viewing the final reports:

    1. The "Manual Results" on the Election page need to be filled in.
    2. The Validate Ballots button on the Ballots page the must be clicked to validate the status of each vote and ballot. This must be done again if any vote is later adjusted.

    Print the "Election Summary" report to provide official results of the election. This report shows the number of votes received by the next person not elected. It also attempts to indicate if there is a tie.

    At an Annual Meeting or Unit Convention, if the community votes to have a full report provided with all names, the suggested report to use is called "All by Votes". Alternatively, the "All by Name" may be desired.

  14. Resolve Tie-breaks

    If a careful review of the results shows that there is a tie or a small difference in the number of votes between last person elected and the next electee, a careful double-check that all paper ballots where correctly entered in TallyJ may be required. (Without TallyJ, a full recount would be needed.) If a printer is available, print the "Ballots Cast" report and compare each ballot on the printed report to the corresponding paper ballot (according to the Ballot Id previously written on the paper ballot).

    If there is a tie, a tie-break vote must be taken unless minority rules apply. Once that is completed, the community list for the people involved in the tie should be updated to set their "Tie Break Resolution" setting. This setting (None, Low, High) is used when needed to break the tie. A person with High priority wins over a person with Low priority. Similarly, Low wins over None. Once the priority has been set for the tied electees, use Validate Ballots again, then view the report again. TallyJ is set to correctly handle 2- or 3-way ties.

    Tie-break votes are typically not tallied using TallyJ, since there is only one or a few names being voted for. "Traditional" paper-based tallying methods should be used, where the votes are sorted into piles by name and counted.

    For Unit Conventions where "the believers receiving the next highest number of votes" are reported, all ties involving these people must be resolved by tie-break unless minority rules apply.

    Some of the reports available are:

    • "Election Summary" - Full report with statistics and the names of the elected
    • "Top 9" - Top 9 electees with no counts shown
    • "Top 9+2 with Count" - Top 9 and next few with number of votes received shown
    • "All by Votes" - All electees and how many votes were cast for them
    • "Votes Cast" - All votes cast and their status. Invalidated votes are shown crossed out.
    • "Ballots Cast" - All ballots cast and their status. Invalidated ballots and votes are shown crossed out. A link is provided to directly edit any of the ballots.
    • "All by Name" - Alphabetical listing of all those receiving a vote, with the number of votes shown.
    • "Summary - Ballots and Votes" - Statistical counts and summary for the election.

    You can print any report with the Print button or CTRL-P. The contents of the report page can also be copied into another program, such as MS Word to be included in existing reports.

    PDF output files can be made by installing a PDF printer driver, and selecting it as the output printer. There are a number of such drivers available. One that seems to work well is available from PDF Creator.

  15. Archive the Election documents

    After the election, many communities store the paper ballots for a year. A copy of the TallyJ folder from the Chief Teller's computer could be copied to a diskette and included in the archives.

  16. Translating TallyJ into other languages

    As of version 1.4, TallyJ can be translated into other languages. Multiple Language Modules can exist side-by-side on the computer and the desired one used in TallyJ. Most translation is done in a copy of the "Phrases_EN.txt" file. Help files can also be copied and translated. All files for translation exist in the main folder or the "Support/Display" folder.

    TallyJ uses the concept of language and culture codes for identifying languages. For example, EN, FR, FR-CA, DE-CH, ES, etc. The "culture code" can also be used to make variant versions of TallyJ.

    Suggested steps for translating:

    1. Copy and edit the Phrases_EN.txt file. More instructions can be found at the bottom of that file. Each language has its own phrases file. The phrases files form the core of TallyJ.
    2. (Optional) Make a copy and translate each file (in the TallyJ folder and in the TallyJ\Support\Display folder) that ends with _EN.htm. The new files should be named according to the language code being translated, e.g. Overview_DE.htm.
